📋 탭 1 — 매출 현황

1
채널 추가
상단의 채널 추가 영역에서 판매 중인 쇼핑몰을 선택하고 "+ 채널 추가"를 클릭합니다. 여러 채널을 한 번에 추가하려면 "전체 채널 일괄 추가"를 사용하세요.
💡 목록에 없는 채널은 "직접 입력..."을 선택하면 입력창이 나타납니다.
2
주차 추가
날짜를 선택하면 해당 날짜가 포함된 주(월~일)가 자동 계산됩니다. 채널을 선택한 후 "+ 주차 추가"를 클릭하세요.
3
실매출액 입력
방법 A (자동) — 채널명 옆 📤 버튼으로 쇼핑몰 정산 엑셀 파일을 업로드하면 주 단위로 자동 집계됩니다.
방법 B (수동) — 실매출액 셀을 클릭해 직접 입력합니다.
💡 주문번호 앞 8자리(yyyymmdd)도 날짜로 자동 인식합니다.
4
저장 및 동기화
주차 추가 바 우측의 💾 저장을 클릭하면 로컬 및 클라우드에 저장됩니다. 상단 우측 ☁️ 동기화 설정 버튼으로 Firebase를 연결하면 기기 간 자동 동기화가 활성화됩니다.
엑셀 파일 날짜/금액이 인식 안 될 때
컬럼명에 "결제완료일", "주문일", "금액" 키워드가 있는지 확인하세요.
새로고침 시 데이터가 사라지나요?
💾 저장 후에는 유지됩니다. Firebase 동기화 연결 시 클라우드에도 자동 저장됩니다.
기기 간 동기화 설정 방법
상단 우측 ☁️ 동기화 설정을 클릭하여 Firebase Realtime Database URL, API Key, Project ID 입력 후 연결하세요. 모든 기기에서 동일한 동기화 키를 사용하면 데이터가 공유됩니다.

🧮 탭 2 — 예상매출 계산

1
요일지수 설정
엑셀 업로드로 자동 계산하거나 직접 입력합니다. 채널별 개별 지수 설정도 가능합니다.
💡 "균등분배"는 14.3%씩, "정규화"는 합계 100%로 자동 조정합니다.
2
월·화·수 실매출 입력
이번 주 월~수요일의 실제 판매액을 입력합니다. 합계와 일평균이 자동 계산됩니다.
3
상황 보정 및 현황표 적용
상황 버튼과 슬라이더로 보정 후, "📋 현황표에 적용"으로 예상매출액을 자동 입력합니다.
☁️ 기기 간 동기화 설정
Firebase Realtime Database를 연결하면 동일한 동기화 키를 사용하는 모든 기기에서 데이터가 실시간으로 공유됩니다.
Firebase Console에서 프로젝트 생성 → Realtime Database 활성화 → 웹 앱 등록 후 아래 항목을 입력하세요.
Firebase 프로젝트 정보
동기화 설정
